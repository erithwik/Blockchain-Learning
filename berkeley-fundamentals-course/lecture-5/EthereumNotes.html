Ethereum Notes

What makes bitcoin so special?

  • Cryptographic identities
  • Blockchain
  • Trustless Consensus
  • No central authority

Bitcoin created a distributed network which is pseudoanonymous due to cryptographic identities, is democratic where decisions are amde through a consensus protocol, is immutable due to all transactions existing in the blockchain, is uncensorable since it cannot be controlled by one party, and is distributed lacking a central point of failure.

Smart Contracts

Code that facilitates, verifies, or enforces the negotiation or execution of a digital contract. A trusted entity must run this code.

Essentially, it is like a regular contract between two groups but this contract is fully enforced by code.

What is Ethereum?

  • It is a decentralized platform designed to run smart contracts. It is like a distributed computer that executes code (everyone runs the code and we come to a consensus on the output), and it is a distributed state machine. It also has ether. It is also account based while Bitcoin is UTXO based.

  • In bitcoin, a state is how much money (how many utxos) each person has, but in Ethereum, each state is how much money each person has alongside what the values are in each of the smart contract's variables, who has ownership on theses non-coin items, etc.

  • Every node executs Ethereum smart contracts, and then comes to a consensus on the new network state.
    • Smart Contracts in Ethereum are like autonomous agents that live inside the Ethereum Network. They react to external world when they are poked by transactions which call specific functions. They have direct control over:
  • Internal Ether Balance
  • Internal Contract State

    • Essentially, the smart contracts handle all the finances given the certain restrictions and algorithms the users give it, so that the users don't have to make any calculations by themselves.

    Purposes of Smart Contracts

  • Store and maintain data

    • Data represents something useful to others or other contracts.

  • Manage contact or relationships between untrusting (or anonymous users)

    • Financial contracts, escrow, insurance, betting. For example, lets say that you are buying something from a person. You put money into the smart contract, but the smart contract doesn't send the money to the other person until it is sure that the other person has met all the requirements to recieve the item (such as finishing all the paperwork or completely sending the item). If the smart contract sees that the vendor is doing something sketchy, then it won't complete the transaction. It is essentially the middleman that's alwyas accurate and doesn't need trusting.

  • Providing functions to other contracts

    • Such as some algorithms that can be useful in other smart contracts. Smart contracts can call each other.

  • Complex authentication

    • Such as we need M of N signatures to do something sorta things.

    EVM Gas and Fees

    Every contract requires gas which feuls the contract execution. Every transaction specifies the startgas -> maximum quanitity of gas it is willing to consume and the gas price -> fee in ether it is willing to pay per unit of gas. So overall consumption of ether = startgas * gasprice. This gas will be paid to the person who runs the smart contract.

    So essentially, when you are purchasing gas, you are essentially purchasing distributed, trustless, computational power.
